# Quick Task 260405-t11: MQTT Admin Page Edit User Dialog + TLS Support

Edit-user dialog with create/edit dual mode and mqtts:// TLS configuration for cloud broker connectivity.

## Task 1: Edit user dialog + page wiring

**Commit:** `0d161d4`

Converted `MqttUserDialog` from create-only to dual create/edit mode:
- Added `editUser` optional prop and `onSaved` callback (renamed from `onCreated`)
- Edit mode pre-fills username (disabled), textName, and role from existing user data
- Password is optional in edit mode with helper text; validated only when non-empty (min 8 chars)
- Submit dispatches PUT to `/api/mqtt/users/:username` in edit mode, POST in create mode
- Dialog title and submit button label change based on mode

Wired edit button into MQTT users table on `page.tsx`:
- Added `editTarget` state to track which user is being edited
- Added Pencil icon button before delete button for non-system accounts
- Create User button clears editTarget before opening dialog
- Dialog close handler clears editTarget

Added i18n keys: `edit`, `save`, `updated`, `passwordOptional`, `passwordMinLength` in both en.json and it.json.

## Task 2: TLS/SSL support for cloud brokers

**Commit:** `33af4a6`

Full-stack TLS support across types, backend, and frontend:
- Extended `IMqttConfig` and `MqttConfigSchema` with `useTls: boolean` and `caCert: string | null`
- Added `use_tls` and `ca_cert` columns to Drizzle schema and SQL CREATE TABLE
- Added ALTER TABLE migration for existing tables lacking the new columns
- Added `mqttUseTls` and `mqttCaCert` env config fallbacks
- MQTT plugin now uses `mqtts://` protocol when TLS enabled, passes CA cert buffer to TLS options
- PUT `/api/mqtt/config` route validates `useTls` and `caCert` fields
- Config form shows TLS toggle switch under new "Security" section; CA cert textarea conditionally renders when TLS is enabled
- Added i18n keys: `securityTitle`, `useTls`, `caCert`, `caCertHelp`
